### Key Integration Points Verified

1. `eaml_parser::parse()` calls `eaml_lexer::lex()` internally — lex diagnostics merge into parser output
2. `eaml_cli::run_pipeline()` calls `parse() → analyze() → generate()` in sequence
3. `all_diagnostics` in CLI aggregates diagnostics from parse + semantic phases
4. `has_fatal` flag from semantic analysis gates codegen (CAP010 prevents Python emission)
5. Generated Python imports `from eaml_runtime import execute_prompt` matching the runtime API surface
6. `Span` type alias chain is consistent across all crates (all resolve to `Range<usize>`)
