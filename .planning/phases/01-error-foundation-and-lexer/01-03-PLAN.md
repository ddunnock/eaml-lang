    Modify the Lexer in `lexer.rs` to fully implement template string scanning. The entry point already exists from Plan 02 (when `DoubleQuote` is encountered, mode switches to TemplateString). This task completes the character-by-character scanning logic.

    **TemplateString mode scanning (`scan_template_string` method):**

    Loop through source bytes starting from the position after the opening `"`:

    1. **Regular text character**: Accumulate into current text span. Continue.

    2. **`\` (backslash)**: Check next character:
       - `n`, `t`, `r`, `"`, `\`: Valid escape. Include in text content. Advance past both characters.
       - Any other char: Emit SYN004 diagnostic ("invalid escape sequence `\\{char}`"). Include both chars in text span. Continue.

    3. **`{` followed by `{`**: This is an escaped brace. Emit any accumulated text up to this point as TmplText, then emit a TmplText token for the literal `{` character. Advance past both `{` characters. Per CONTEXT.md: "lexer handles brace escaping: `{{` and `}}` are converted to text fragments containing literal `{` and `}`".

    4. **`}` followed by `}`**: Same as above but for `}`. Emit TmplText for literal `}`. Advance past both.

    5. **`{` (single, not followed by `{`)**: End of text fragment. Emit accumulated TmplText (if any). Emit TmplInterpStart token. Set mode to `Interpolation { brace_depth: 1 }`. Return to main tokenize loop (which will use logos for expression tokens).

    6. **`"` (closing quote)**: End of template string. Emit accumulated TmplText (if any). Emit TmplEnd. Set mode back to Normal. Return to main loop.

    7. **EOF**: Unterminated string. Emit SYN002 diagnostic ("unterminated string literal") pointing to the opening `"` position. Emit accumulated TmplText (if any). Emit TmplEnd for recovery. Set mode to Normal.

    8. **End of line (`\n`) in unterminated interpolation context**: If we're recovering from an unclosed interpolation (SYN045), stop scanning at the newline, close the template.

    **Interpolation mode handling (in main tokenize loop):**

    When mode is `Interpolation { brace_depth }`:
    - Use logos to scan tokens normally
    - When logos yields `LBrace`: emit LBrace token, increment brace_depth
    - When logos yields `RBrace`: decrement brace_depth. If brace_depth reaches 0: emit TmplInterpEnd, set mode to `TemplateString { brace_depth: 0 }`, return to template scanning. Otherwise: emit RBrace token.
    - When logos yields `DoubleQuote`: This is a nested string inside interpolation. Handle recursively (push template mode).
    - On EOF with brace_depth > 0: emit SYN045 diagnostic ("unclosed template string interpolation"), pointing to the opening `{`. Emit TmplInterpEnd and TmplEnd for recovery.
    - On end-of-line recovery: per CONTEXT.md, "unterminated interpolation (missing closing brace): emit SYN error, recover at end of line, continue lexing".

    **Python bridge scanning in lexer.rs:**

    Implement the `scan_python_bridge` method (skeleton from Plan 02):

    When the tokenizer encounters `python` keyword followed by `%{` (the logos RawToken::PercentLBrace):
    1. Emit `TokenKind::KwPythonBridge` token for the `python` keyword (not KwPython -- this distinguishes bridge context).
    2. Record the start position (position of `python` keyword for error pointing).
    3. Skip past `%{` (do not emit as a separate token).
    4. Set mode to `PythonBridge`.
    5. Begin character-by-character scanning of bridge content.

    **PythonBridge mode scanning:**
    ```
    content_start = current position (after %{)
    loop:
        if at EOF:
            emit SYN046 diagnostic("unclosed python bridge block")
                with span pointing back to the opening "python %{"
                with hint: "expected closing }% at the start of a line"
            emit PythonBlock token with content from content_start to EOF
            set mode to Normal
            return

        if at start of line (or only whitespace since last newline):
            if looking_at("}%"):
                content = source[content_start..current_pos]
                advance past "}%"
                emit PythonBlock token with captured content
                set mode to Normal
                return

        advance 1 byte
    ```

    **"Start of line" detection for `}%`:**
    Track whether we're at the start of a line. A position is "at line start" if:
    - It's position 0 in the bridge content, OR
    - The previous character was `\n`, OR
    - All characters since the last `\n` (or content start) are whitespace (`' '` or `\t`)

    This means `}%` in `f"{value}% done"` is NOT at line start (there's non-whitespace content `f"{value}` before it on that line), so it's not recognized as a closing delimiter.

    But `  }%` on its own line IS recognized (only whitespace before `}%`).

    **Error collapsing (post-processing):**

    After the main tokenization loop completes, post-process the diagnostics list:
    - Scan for adjacent SYN001 ("unexpected character") diagnostics where the spans are contiguous (end of one == start of next).
    - Collapse them into a single diagnostic with a span covering the full range.
    - Update the message to "unexpected characters" (plural) with the combined span.
    - Per CONTEXT.md: "Consecutive identical errors at adjacent positions: collapse into one diagnostic spanning the range (e.g., 'unexpected characters at 5:1-5:10' instead of 10 separate errors)".
