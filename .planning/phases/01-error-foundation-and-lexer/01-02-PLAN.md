    **lexer.rs**: Build the wrapper Lexer struct that drives logos and produces the public Token stream.

    ```rust
    use crate::logos_lexer::RawToken;
    use crate::token::{Token, TokenKind, Span};
    use crate::intern::Interner;
    use eaml_errors::{Diagnostic, ErrorCode, Severity};

    #[derive(Debug, Clone, Copy, PartialEq)]
    enum LexerMode {
        Normal,
        TemplateString { brace_depth: u32 },
        Interpolation { brace_depth: u32 },
        PythonBridge,
    }

    pub struct LexOutput {
        pub tokens: Vec<Token>,
        pub diagnostics: Vec<Diagnostic>,
        pub interner: Interner,
    }

    pub struct Lexer<'src> {
        source: &'src str,
        pos: usize,
        mode: LexerMode,
        interner: Interner,
        diagnostics: Vec<Diagnostic>,
        tokens: Vec<Token>,
    }
    ```

    The Lexer does NOT use logos as an iterator directly. Instead, it processes the source character by character in some modes and delegates to logos in Normal mode.

    **Normal mode flow:**
    1. Normalize source to LF (replace all `\r\n` with `\n` and lone `\r` with `\n`) at construction time -- this is the recommended approach per RESEARCH.md Pitfall 5. All spans are post-normalization.
    2. Create a `logos::Lexer<RawToken>` from the normalized source.
    3. Iterate logos tokens. For each:
       - Map RawToken variants to TokenKind variants (keywords, operators, literals map 1:1)
       - For `RawToken::Ident`: intern the slice via `self.interner.intern(slice)`, emit `TokenKind::Ident(spur)`
       - For `RawToken::IntLit` / `RawToken::FloatLit`: emit `TokenKind::IntLit` / `TokenKind::FloatLit` with span (value extracted from source span later by parser)
       - For `RawToken::DoubleQuote`: switch to TemplateString mode and emit `TokenKind::TmplStart`. Then hand-scan the string content (see below).
       - For `RawToken::PercentLBrace`: check if previous meaningful token was KwPython. If yes, switch to PythonBridge mode. If no, emit as two separate tokens or error.
       - For `Err(())`: emit SYN001 diagnostic ("unexpected character") at the span, skip the byte, continue.
    4. At end of input, emit `TokenKind::Eof`.

    **String/template scanning (in TemplateString mode):**
    When entering a double-quoted string, hand-scan character by character from current position:
    - Accumulate text characters into a span for TmplText
    - On `\` (backslash): check next char. If `n`, `t`, `r`, `"`, `\`: valid escape, include in text span. Otherwise: emit SYN004 diagnostic ("invalid escape sequence `\\{char}`"), include the backslash and char in text.
    - On `{` (single): emit accumulated TmplText (if any), emit TmplInterpStart, set mode to Interpolation { brace_depth: 1 }, and return to logos-based scanning for the expression tokens.
    - On `{{`: emit TmplText containing literal `{` character (one byte span for display, but the text content is `{`).
    - On `}}`: emit TmplText containing literal `}`.
    - On `"`: emit accumulated TmplText (if any), emit TmplEnd, return to Normal mode.
    - On newline (`\n`): normalize to LF in text content (already done by source normalization).
    - On EOF: emit SYN002 diagnostic ("unterminated string literal"), emit TmplEnd to close the token, return to Normal mode.

    **Interpolation mode** (inside `{...}` in template strings):
    - Use logos to scan tokens as in Normal mode
    - Track brace depth: `{` increments, `}` decrements
    - When depth reaches 0 on `}`: emit TmplInterpEnd, return to TemplateString mode
    - If EOF reached with depth > 0: emit SYN045 diagnostic ("unclosed template string interpolation")

    **Error recovery:**
    - Unexpected character (logos Err): emit SYN001 diagnostic with span covering the bad byte, skip 1 byte, continue.
    - Unterminated string: emit SYN002, recover at end of line (first `\n`) or EOF.
    - Unterminated block comment: This is tricky -- logos handles block comments via skip. If logos fails to skip a `/*` because there's no closing `*/`, logos will emit the `/` and `*` as errors. To detect this properly: after lexing completes, check if there are adjacent SYN001 errors starting with `/` and `*` -- OR, pre-scan the source for unclosed `/*` before lexing. Use the simpler approach: let logos handle it, and if `/` and `*` appear as errors, collapse them.
    - Consecutive identical errors at adjacent positions: After lexing, post-process diagnostics to collapse adjacent SYN001 errors into a single diagnostic spanning the range (per CONTEXT.md decision).

    **Public API in lib.rs:**
    ```rust
    pub fn lex(source: &str) -> LexOutput {
        let mut lexer = Lexer::new(source);
        lexer.tokenize();
        lexer.into_output()
    }
    ```
