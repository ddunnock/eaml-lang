    **Create `crates/eaml-semantic/src/cap_checker.rs`:**

    **Known capabilities registry (hardcoded for v0.1):**
    ```rust
    const KNOWN_CAPABILITIES: &[&str] = &[
        "json_mode", "streaming", "function_calling", "vision", "code_interpreter",
    ];
    ```

    **Public API:**
    ```rust
    pub fn check(
        program: &Program,
        ast: &Ast,
        interner: &Interner,
        symbols: &SymbolTable,
        type_annotations: &TypeAnnotations,
        source: &str,
        diags: &mut DiagnosticCollector,
    )
    ```

    **Step 1: Validate individual capability declarations.**
    For each model declaration:
    - Check each cap in `model.caps` against `KNOWN_CAPABILITIES`. If not found, emit `ErrorCode::Cap001` warning ("unknown capability '{cap_name}'").
    - Check for duplicates within `model.caps`. If duplicate found, emit `ErrorCode::Cap002` warning ("duplicate capability '{cap_name}'").

    For each prompt with `requires` clause:
    - Same checks: unknown caps -> CAP001, duplicate caps -> CAP002.

    **Step 2: Capability subset checking.**
    Determine prompt-to-model bindings:
    - Walk all agent declarations. For each agent with `AgentField::Model(model_spur, _)`:
      - Resolve model_spur to a ModelDecl via symbol table.
      - Collect model's capabilities as `HashSet<Spur>`.
      - (For v0.1, the agent doesn't directly list prompts, so check ALL prompts with requires clauses against this model.)
    - If no agents exist, check each prompt's requires against ALL models.
    - For each (prompt, model) pair where prompt has requires:
      - Compute missing = prompt.requires.caps - model.caps (set difference using Spur comparison)
      - If missing is non-empty, emit `ErrorCode::Cap010` with FATAL severity:
        ```
        message: "model '{model_name}' is missing required capabilities. Required: [{required}]. Provided: [{provided}]. Missing: [{missing}]"
        span: requires.span
        label: "required here"
        secondary_label: (model.span, "model declared here")
        ```

    **Step 3: json_mode + string return type check (CAP020).**
    For each prompt with requires clause containing "json_mode":
    - Check the prompt's return type. If it resolves to `ResolvedType::Primitive("string")`, emit `ErrorCode::Cap020` warning ("json_mode is typically used with structured return types, not string").
