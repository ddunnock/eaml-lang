    **Create `crates/eaml-semantic/src/type_checker.rs`** with the following structure:

    **Public API:**
    ```rust
    pub struct TypeAnnotations {
        pub type_exprs: HashMap<TypeExprId, ResolvedType>,
    }

    #[derive(Debug, Clone)]
    pub enum ResolvedType {
        Primitive(String),  // "string", "int", "float", "bool", "null"
        Schema(SchemaDeclId),
        Array(Box<ResolvedType>),
        Optional(Box<ResolvedType>),
        LiteralUnion(Vec<String>),
        Error,
    }

    pub fn check(
        program: &Program,
        ast: &Ast,
        interner: &Interner,
        symbols: &SymbolTable,
        source: &str,
        diags: &mut DiagnosticCollector,
    ) -> TypeAnnotations
    ```

    **Type expression validation (`check_type_expr`):**
    - `TypeExpr::Named(spur, span)` -> if not `symbols.is_known_type(spur)`, emit `ErrorCode::Typ010` ("unknown type '{name}'"). Return `ResolvedType::Error`. Otherwise return `ResolvedType::Primitive(name)` or `ResolvedType::Schema(id)`.
    - `TypeExpr::Bounded { base, params, span }` -> validate:
      1. Base must be "string", "int", or "float" -- else emit `ErrorCode::Typ032` ("type '{base}' does not accept bounded parameters")
      2. Normalize positional to named: if all params have `name: None`, first=min, second=max. If params have `name: Some(spur)`, use named form.
      3. If any named param is not "min" or "max" (for int/float) or "minLen"/"maxLen" (for string), emit `ErrorCode::Sem030` ("unknown bounded parameter name '{name}'")
      4. Parse numeric values from `source[param.value_span.clone()]` -- use `str::parse::<f64>()` for float, `str::parse::<i64>()` for int
      5. If min > max, emit `ErrorCode::Typ030` ("lower bound ({min}) exceeds upper bound ({max})")
      6. For string bounds: if any value is negative, emit `ErrorCode::Typ031` ("invalid string length bound: must be non-negative")
    - `TypeExpr::Array(inner, _)` -> recurse, wrap in `ResolvedType::Array`
    - `TypeExpr::Optional(inner, _)` -> recurse, wrap in `ResolvedType::Optional`
    - `TypeExpr::LiteralUnion { members, span }` -> extract literal values from `source[member_span.clone()]`, strip quotes, check for duplicates. Duplicate -> emit `ErrorCode::Typ040` warning ("duplicate literal union member '{value}'")
    - `TypeExpr::Grouped(inner, _)` -> recurse
    - `TypeExpr::Error(_)` -> return `ResolvedType::Error`

    **Schema validation (`check_schema`):**
    - Check for duplicate field names within schema -> emit `ErrorCode::Sem020` ("duplicate field name '{name}' in schema '{schema_name}'")
    - Validate each field's type_expr via `check_type_expr`
    - Check if schema name shadows primitive type -> emit `ErrorCode::Typ001` warning ("schema '{name}' shadows built-in type")

    **Prompt validation (`check_prompt`):**
    - Check prompt body has at least one `PromptField::User(_)` -> else emit `ErrorCode::Sem025` ("prompt body missing required 'user:' field")
    - Validate return type
    - Validate parameter types

    **Tool validation (`check_tool`):**
    - Check `ToolBody::Empty` -> emit `ErrorCode::Sem040` ("tool body has no implementation")
    - Validate return type and parameter types
    - Python bridge bodies are OPAQUE -- do NOT validate

    **Model validation (`check_model`):**
    - Extract provider string from `model.provider` template string. If the first text part is not one of "anthropic", "openai", "ollama", emit `ErrorCode::Pyb010` warning ("unknown provider '{value}'")

    **Expression validation (`check_expr`) -- minimal for this plan:**
    - `Expr::BinaryOp { left, op, right, .. }` where op is comparison (Eq, NotEq, Lt, Gt, LtEq, GtEq): check if left or right is also a comparison BinaryOp. If so, emit `ErrorCode::Sem060` ("chained comparison detected") with hint "use explicit grouping: (a == b) && (b == c)"
    - `Expr::Error(_)` -> skip

    **Update `crates/eaml-semantic/src/lib.rs`:**

    **Add template scope validation to type_checker.rs:**

    In `check_prompt()`, after validating structure, build a `Scope` containing:
    1. All prompt parameters (`prompt.params` -- each param's `name` Spur)
    2. All top-level let bindings (from `symbols` where `SymbolKind::Let`)

    Walk each `PromptField::User(ts)` and `PromptField::System(ts)` template string. For each `TemplatePart::Interpolation(expr_id, span)`:
    - Call `check_template_expr(expr_id, &scope, ast, interner, diags)` which:
      - `Expr::Ident(spur, span)` -> check `scope.contains(spur)` OR `symbols.get(spur).is_some()` (for top-level declarations). If not found, emit `ErrorCode::Res001` with "undefined variable '{name}' in template interpolation" and "did you mean?" hint.

      - Other expr types -> recurse as needed, checking idents at leaves

    **In `check_tool()` for native bodies**, build scope with tool parameters and validate expressions similarly. For `ToolBody::PythonBridge` -- do NOT validate (opaque).

    **Sequential let bindings in prompt/tool expressions:**
    If prompt/tool body contains `Expr::Let { name, value, .. }` expressions:
    - Resolve `value` against current scope
    - THEN add `name` to scope (sequential: value cannot reference the binding being defined)
